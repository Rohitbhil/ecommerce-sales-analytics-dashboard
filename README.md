# 📊 E-Commerce Sales Analytics Dashboard

This project analyzes 99K+ orders, 96K+ customers, and 100K+ payment records from a Brazilian e-commerce dataset to generate actionable business insights using SQL (PostgreSQL) and Power BI.

---

## 🔧 Tech Stack
- SQL (PostgreSQL)
- Power BI
- DAX
- Data Modeling (Star Schema)
- Business Intelligence & Data Visualization

---

## 📌 Project Overview

The objective of this project was to design an end-to-end Business Intelligence solution to analyze revenue performance, customer behavior, product trends, and payment insights.

Data was extracted and validated using SQL, then modeled in Power BI using a star-schema structure to ensure accurate aggregation and category-level reporting.

---

## 📈 Key Metrics
- ₹16M+ Total Revenue
- 99K+ Orders Analyzed
- 96K+ Customers
- 100K+ Payment Records

---

## 📊 Dashboard Features

### 1️⃣ Executive Overview
- KPI Cards (Revenue, Orders, Customers, AOV)
- Revenue Trend Over Time
- Top 10 Product Categories by Revenue
- Geographic Revenue Distribution
- Payment Method Analysis
- Order Status Distribution

### 2️⃣ Customer & Product Insights
- Top 10 Customers by Revenue
- Category-Level Average Order Value (AOV)
- Product-Level Sales Analysis
- Granularity mismatch resolution between order-level and item-level data

---

## 🧠 Technical Highlights
- Performed SQL joins and aggregations across multiple relational tables.
- Designed a star-schema data model for optimized reporting.
- Developed DAX measures using SUMX, DISTINCTCOUNT, and DIVIDE.
- Identified and resolved data granularity mismatches to ensure accurate revenue calculations.

---

## 📂 Dataset

This project uses the Brazilian E-Commerce Public Dataset (Olist), a real-world anonymized transactional dataset.

[Olist Brazilian E-Commerce Dataset (Kaggle)](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)
